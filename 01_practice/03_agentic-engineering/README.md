# 💬 채팅방 요약 도우미

채팅방에서 내보낸 **CSV**를 올리면, 원하는 **기간**을 골라 **요약 · 핵심 주제 · 결정 사항 · 액션 아이템**을 만들어 주는 아주 간단한 프로토타입 MVP입니다. 채팅이 너무 많아 다 읽기 어려울 때, 놓친 구간만 빠르게 따라잡는 용도예요.

- **CSV 파싱·통계·기간 필터는 브라우저**에서 처리합니다.
- **요약만** 작은 Node 백엔드를 거쳐 **OpenAI**로 전달합니다.
- **API 키는 서버(`.env`)에만** 있고 브라우저로는 절대 나가지 않습니다.

## 실행 방법

```bash
cp .env.example .env      # OPENAI_API_KEY 채우기
npm install
npm start
# 브라우저에서 http://localhost:3000 접속
```

처음이라면 화면의 **"샘플 데이터로 먼저 둘러보기"** 버튼으로 동작을 확인해 보세요. (통계 미리보기는 키 없이도 동작하고, **요약 생성**에는 유효한 `OPENAI_API_KEY`가 필요합니다.)

## 환경 변수 (`.env`)

| 변수 | 설명 |
|------|------|
| `OPENAI_API_KEY` | OpenAI API 키 (https://platform.openai.com/api-keys) |
| `OPENAI_MODEL` | 요약에 쓸 모델. 기본 `gpt-4o` |
| `PORT` | 서버 포트. 기본 `3000` |

## 입력 CSV 형식

`Date,User,Message` 컬럼을 기본으로 인식합니다 (KakaoTalk 내보내기 형식). 컬럼명이 달라도 자동 매칭을 시도하고, 못 찾으면 `[날짜, 사용자, 메시지]` 순서로 추정합니다.

```csv
Date,User,Message
2026-06-18 09:02:11,"민수","발표 순서 정해요"
2026-06-18 09:06:15,"민수","발표는 6/26 금요일 저녁 8시로 확정"
```

- UTF-8(BOM) / EUC-KR 인코딩을 자동 감지합니다.
- 입장·퇴장 같은 시스템 메시지는 통계·요약에서 자동 제외됩니다.

## 동작 방식

1. CSV 업로드 → 메시지 파싱, 기본 통계(기간·메시지 수·참여자) 표시
2. 기간 선택 (오늘 / 최근 3일 / 최근 7일 / 전체 / 커스텀 — 가장 최근 메시지 기준)
3. **요약 생성** → 선택 구간을 `/api/summarize`로 전송 → OpenAI가 4개 섹션으로 정리

대화량이 너무 많으면(약 30만 자 초과) 기간을 좁히라고 안내합니다.
